This specification defines an API that lets webapps setup geographic boundaries around specific locations and then receive notifications when the hosting device enters or leaves those areas.

Introduction

The Geofencing API lets webapps setup geographic boundaries around specific locations and then receive notifications when the hosting device enters or leaves those areas. While it would be possible to implement something similar using the Geolocation API [[!GEOLOCATION-API]], there are a few differences that could make this API a better choice:

  • Because of the limited API surface of the Geofencing API, user agents can implement the API in a more (power) efficient way than could be achieved by regularly checking the current geographic position with the Geolocation API.
  • The Geofencing API is built around Service Workers. This makes it possible for a webapp to receive notifications from the Geofencing API even after the user has closed the webapp.

this.ongeofenceenter = function(event) {
  console.log(event.registration.id);
  console.log(event.registration.region.name);

  // If this is not a registration of interest anymore, unregister it.
  if (event.registration.region.name !== "myfence") {

  console.log(event.registration.region.name);

This specification defines conformance criteria that apply to a single product: the user agent that implements the interfaces that it contains.

Implementations that use ECMAScript to implement the APIs defined in this specification MUST implement them in a manner consistent with the ECMAScript Bindings defined in the Web IDL specification [[!WEBIDL]].

Terminology

The terms event handler, event handler event type, queue a task, and fire a simple event are defined in [[!HTML5]].

The types Promise, and RangeError are defined in [[!ECMASCRIPT]].

EventInit, DOMException, InvalidStateError, NotFoundError, QuotaExceededError, SyntaxError, and steps for constructing events are defined in [[!DOM]].

The terms service worker, service worker registration, installing worker, waiting worker, and active worker, and the types ServiceWorkerRegistration, ServiceWorkerGlobalScope, ExtendableEvent, and ExtendableEventInit are defined in [[!SERVICE-WORKERS]].

The type Position is defined in [[!GEOLOCATION-API]].

The term webapp refers to a Web application, i.e. an application implemented using Web technologies, and executing within the context of a Web user agent, e.g. a Web browser or other Web runtime environment.

The term geofence registration is used to refer to one specific registration of a geographic region as a geofence. A geofence registration is a tuple consisting of a geographic region, registration ID and a include position flag.

The term active geofences is used to refer to the collection of geofence registrations associated with a particular service worker registration, that are currently being monitored by the user agent for breach events.

A geofence registration is said to be breached if the current geographical location changed from being inside the geographic region to outside (a leave event), or vice versa (an enter event).

Security and privacy considerations

The same security and privacy considerations that apply to the Geolocation API [[!GEOLOCATION-API]] also apply to this API. Furthermore since this API effectively gives access to geographic location information after a user has stopped interacting with a webapp, a few other considerations should be taken into account.

API Description

Extensions to the ServiceWorkerRegistration interface

The Service Worker specification defines a ServiceWorkerRegistration interface, which this specification extends.

readonly attribute GeofenceController geofencing

The geofencing attribute exposes a GeofenceController, which has an associated service worker registration represented by the ServiceWorkerRegistration on which the attribute is exposed.

GeofenceController interface

The GeofenceController interface defines operations that enable webapps to establish access to geofencing services.

Promise<GeofenceRegistration> registerRegion(GeofenceRegion initialRegion, optional GeofenceRegisterOptions options)
Promise<sequence<GeofenceRegistration>> getRegisteredRegions(optional GeofenceRegistrationQueryOptions options)
Promise<GeofenceRegistration> getRegisteredRegion(DOMString id)

The registerRegion method when invoked MUST run the following steps:

  1. Let promise be a new Promise.
  2. Run the following steps in parallel:
    1. If the total number of active geofences for all service worker registrations in the current origin is more than a user agent defined limit, reject promise with a DOMException whose name is "QuotaExceededError" and terminate these substeps. A user agent SHOULD allow at least 20 active geofences for an origin.
    2. Ask the user whether they allow the webapp to use geofences, unless a prearranged trust relationship applies or the user has already granted or denied permission explicitly for this webapp.
    3. If not granted, reject promise with a DOMException whose name is "PermissionDeniedError" and terminate these substeps.
    4. Let registration be a new geofence registration.
    5. Set the registration ID of the registration to a value uniquely identifying this registration. This ID MUST be unique across all registrations for this particular origin.
    6. Set the geographic region of the registration to a GeofenceRegion instance of the same type and with the same attributes as the initialRegion passed to registerRegion.
    7. Set the include position flag of the registration to options.includePosition, or false if no options were specified.
    8. Add registration to the active geofences of the service worker registration associated with the webapp.
    9. If the current geographic position is inside the newly added region, fire a geofenceenter event.
    10. Resolve promise with registration.
  3. Return promise.

Somehow mention that the region that is saved as part of the registration can be slightly different from the region passed to registerRegion. An implementation may adjust parameters to be in range of what is possible, or otherwise modify the region.

The iOS API takes a slightly different approach when permission is denied. It instead always treats registrations as a success, but won't actually track geofences/trigger events unless the user has granted permission.

If the includePosition attribute is set to true, GeofenceEvents for this registration will have a position attribute. When set to false, the position attribute will always be undefined.

The getRegisteredRegions method when invoked MUST run the following steps:

  1. Let promise be a new Promise.
  2. Run the following steps in parallel:
    1. Let registrations be a new sequence.
    2. For each geofence registration in the active geofences associated with this service worker registration, run the following substeps:
      1. If options is passed, has a non-null name attribute, and that name is not equal to the name of the registration, skip the rest of these substeps and continue with the next registration.
      2. Let registration be a new GeofenceRegistration instance representing the geofence registration.
      3. Append registration to registrations.
    3. Resolve promise with registrations.
  3. Return promise.

The getRegisteredRegion method when invoked MUST run the following steps:

  1. Let promise be a new Promise.
  2. Run the following steps in parallel:
    1. For each geofence registration in the active geofences associated with this service worker registration, run the following substeps:
      1. If the registration ID of the current registration is not equal to the passed in id, skip the rest of these substeps and continue with the next registration.
      2. Let registration be a new GeofenceRegistration instance with attributes equal to those of the geofence registration.
      3. Resolve promise with registration and abort the remainder of these substeps.
    2. If promise was not resolved, reject promise with a DOMException whose name is "NotFoundError".
  3. Return promise.

Maybe instead of reject with NotFoundError this should resolve with null? Both have pros and cons.

GeofenceRegistration interface

An instance of the GeofenceRegistration interface represents a geofence registration.

readonly attribute DOMString id
readonly attribute GeofenceRegion region
Promise<undefined> unregister()

When getting the id attribute, the user agent MUST return the registration ID of the geofence registration. This ID is generated by the user agent and MUST be unique among all current geofence registrations for the same origin, but can be the same as an ID for a geofence registration at a different origin. This ID MUST NOT change as long as this registration exists. The generated ID SHOULD NOT be the same as an ID for any geofence registration that existed before at the same origin.

When getting the region attribute, the user agent MUST return the geographic region of this geofence registration.

The unregister method when invoked MUST run the following steps:

  1. Let promise be a new Promise.
  2. Run the following steps in parallel:
    1. If this geofence registration is not currently in the active geofences associated with a service worker registration, reject promise with a DOMException whose name is "InvalidStateError".
    2. Remove this geofence registration from the set of active geofences associated with the current service worker registration. No more events related to this registration will be fired after this.
    3. Resolve promise with undefined.
  3. Return promise.

GeofenceRegion interface

readonly attribute DOMString name
DOMString? name

The name attribute MUST return the value it was initialized to. When the object is created, this attribute MUST be set to the value of the name property in the GeofenceRegionInit dictionary, or an empty string if that property wasn't set. A user agent MAY impose limits on the maximum size of the name attribute. If this limit is exceeded the constructor MUST throw a RangeError. If a user agent imposes a limit on the size of the name attribute, this limit SHOULD allow for at least 100 characters.

CircularGeofenceRegion interface

readonly attribute GeolocationPoint center
readonly attribute double radius

The center attribute MUST return the value it was initialized to. When the object is created, this attribute MUST be set to a GeolocationPoint instance with its latitude and longitude properties set to the same values as those properties in the CircularGeofenceRegionInit dictionary. This represents the center of the circular region. Latitude must be between -90 and 90 inclusive. Longitude must be between -180 and 180 inclusive. If either of these properties is outside these ranges, the constructor will throw a RangeError.

The radius attribute MUST return the value it was initialized to. When the object is created, this attribute MUST be set to the value of the radius property in the CircularGeofenceRegionInit dictionary. This represents the radius of the circular region in meters.

The latitude property of a GeolocationPoint dictionary represents the latitude in circular degrees of a point.

The longitude property of a GeolocationPoint dictionary represents the longitude in circular degrees of a point.

The geographic coordinate reference system used by the attributes in this API is the World Geodetic System (2d) [[!WGS84]]. No other reference system is supported.

The downside of doing error checks in the CircularGeofenceRegion constructor is that they will be actual exceptions instead of promise rejections when calling register. Would it make sense to move the validity checks to register?

Events

The Service Worker specification defines a ServiceWorkerGlobalScope interface, which this specification extends.

attribute EventHandler ongeofenceenter
attribute EventHandler ongeofenceleave
attribute EventHandler ongeofenceerror

The ongeofenceenter attribute is an event handler whose corresponding event handler event type is geofenceenter.

The ongeofenceleave attribute is an event handler whose corresponding event handler event type is geofenceleave.

The ongeofenceerror attribute is an event handler whose corresponding event handler event type is geofenceerror.

The geofenceenter and geofenceleave events

The GeofenceEvent interface represents a geofence being breached.

readonly attribute GeofenceRegistration registration
readonly attribute Position? position

Upon detecting a breach of a geofence registration, the user agent MUST run the following steps to fire a geofence event:

  1. If the Service Worker associated with the webapp is not running, start it.
  2. Let scope be the ServiceWorkerGlobalScope of the Service Worker associated with the webapp.
  3. Let event be a new GeofenceEvent, whose registration attribute is a new GeofenceRegistration instance representing the geofence registration that was breached.
  4. If the registrations includePosition is true, set event.position to the current geographical position.
  5. Let eventName be geofenceenter or geofenceleave, corresponding to the type of breach event being processed.
  6. Queue a task to fire event as a simple event named eventName at scope.

The user agent MAY wait with firing a geofence event until some time has passed after the breach was detected, or until the for some time or distance after detecting a breach of a registered geofence before firing an event to make sure that the geofence really was breached.

This needs more work. I'm really not sure what attributes make sense to expose here. Also I'm not sure if is desirable/okay for all errors to be fatal/cause the geofence to be unregistered.

Upon detecting some situation in which the user agent won't be able to detect future breached of a geofence registration, the user agent MUST run the following steps:

  1. If the Service Worker associated with the webapp is not running, start it.
  2. Let scope be the ServiceWorkerGlobalScope of the Service Worker associated with the webapp.
  3. Let event be a new GeofenceErrorEvent, whose registration attribute is a new GeofenceRegistration instance representing the geofence registration for which an error was detected.
  4. Let event.code be an error code.
  5. Let event.message be a descriptive message for the error that occured.
  6. Remove the geofence registration for which an error was detected from the set of active geofences associated with the current service worker registration.
  7. Queue a task to fire event as a simple event named geofenceerror at scope.
